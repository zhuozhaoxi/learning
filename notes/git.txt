git merge --no-ff [branch]

git push origin [branch]

git checkout -b [newBranch] => git branch [newBranch] & git checkout [newBranch]


代码回滚（已经push）
方式一：使用revert
撤销上一次提交
git revert HEAD
git push origin master

方式二：使用reset
git reset --hard HEAD^
git push origin master -f

revert是放弃指定提交的修改，但是会生成一次新的提交，需要填写提交注释，以前的历史记录都在；
reset是指将HEAD指针指到指定提交，历史记录中不会出现放弃的提交记录。

代码回滚（本地的某次commit）
git reset [--hard|soft|mixed|merge|keep] [commit|HEAD] -- [file]

还原本地代码到当前版本
git checkout -- [file]

显示本地和origin/master分支的diff文件名
git diff origin/master HEAD --name-only

显示本地和origin/master分支的diff内容
git diff origin/master HEAD


查看本地已经commit，但未push的版本
git cherry -v

查看版本中的具体代码
git show xxxxx_version_id

删除本地分支
git branch -d <BranchName>

删除远程分支
git push origin --delete <BranchName>
